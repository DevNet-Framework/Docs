Relationships

In a relational database, a relationship defines how two entities relate to each other, represented by a foreign key constraint, that store the principal key values for the related entity

  • Principal entity: This is the entity that contains the primary/alternate key properties. Sometimes referred to as the 'parent' of the relationship.
  • Dependent entity: This is the entity that contains the foreign key properties. Sometimes referred to as the 'child' of the relationship.
  • Navigation property: Navigation property is a property defined on the principal or dependent entity that references the related entity. and there are tow types of Navigation property:
    • Collection navigation property: A navigation property that contains references to many related entities.
    • Reference navigation property: A navigation property that holds a reference to a single related entity.

One to Many

The most common pattern for relationships is to have navigation properties defined on both ends of the relationship, the Principal entity has a collection navigation property of dependent entity, and the dependent entity class has a reference navigation property and a foreign key property the Principal entity class.

Principal Entity Class

The following class represent an entity of type Group as a principal entity, it has a property "Id" as primary key, and a collection navigation property Students of type IList, at the runtime it will be initialized by a collection of related entities of type Student. 


                  <?php

                  namespace Application\Models;

                  use Artister\Data\Entity\IEntity;
                  use Artister\system\Collections\IList;

                  class Group implements IEntity
                  {
                     private int $Id;
                     private string $Name;
                     private string $Class;

                     // navigation property
                     private IList $Students;

                     public function __get(string $name){return $this->$name;}

                     public function __set(string $name, $value){$this->$name = $value;}
                  }
Dependent entity class

The following class represent an entity of type Group as a principal entity, it has a property "Id" as primary key, and a collection navigation property Students of type IList, at the runtime it will be initialized by a collection of related entities of type Student. 


                  <?php

                  namespace Application\Models;

                  use Artister\Data\Entity\IEntity;

                  class Student implements IEntity
                  {
                     private int $Id;
                     private int $GroupId;
                     private string $Name;
                     private int $Age;

                     // navigation property
                     private Group $Group;

                     public function __get(string $name){return $this->$name;}

                     public function __set(string $name, $value){$this->$name = $value;}
                  }

Model configuration: 


                  <?php

                  namespace Application\Models;

                  use Artister\Data\Entity\EntityContext;
                  use Artister\Data\Entity\EntityModelBuilder;
                  use Artister\Data\Entity\EntitySet;

                  class EntityManager extends EntityContext
                  {
                     private EntitySet $Groups;
                     private EntitySet $Students;

                     public function __construct()
                     {
                        $this->Groups = $this->Set(Group::class);
                        $this->Students = $this->Set(Student::class);
                     }

                     public function onModelCreate(EntityModelBuilder $builder)
                     {  
                        $builder->entity(Student::class)
                                 ->hasForeignKey('AuthorId', Group::class)
                                 ->hasOne('Group', Group::class);

                        $builder->entity(Group::class)->hasMany('Studens', Student::class);
                     }
                  }

One to one

One to one relationships have a reference navigation property on both sides. They follow the same conventions as one-to-many relationships, but a unique index is introduced on the foreign key property to ensure only one dependent is related to each principal.

Principal Entity Class

                  <?php

                  namespace Application\Models;

                  use Artister\Data\Entity\IEntity;

                  class Author implements IEntity
                  {
                     private int $Id;
                     private string $Name;

                     // navigation property
                     private Picture $Picture;

                     public function __get(string $name){return $this->$name;}

                     public function __set(string $name, $value){$this->$name = $value;}
                  }
Dependent entity class

                  <?php

                  namespace Application\Models;

                  use Artister\Data\Entity\IEntity;

                  class Transcript implements IEntity
                  {
                     private int $Id;
                     private int $StudentId;
                     private float $Score;

                     // navigation property
                     private Student $Student;

                     public function __get(string $name){return $this->$name;}

                     public function __set(string $name, $value){$this->$name = $value;}
                  }

Model configuration: 


                  <?php

                  namespace Application\Models;

                  use Artister\Data\Entity\EntityContext;
                  use Artister\Data\Entity\EntityModelBuilder;
                  use Artister\Data\Entity\EntitySet;

                  class EntityManager extends EntityContext
                  {
                     private EntitySet $Posts;
                     private EntitySet $Pictures;

                     public function __construct()
                     {
                        $this->Posts = $this->Set(Post::class);
                        $this->Pictures = $this->Set(Picture::class);
                     }
                     
                     public function onModelCreate(EntityModelBuilder $builder)
                     {
                        $builder->entity(Picture::class)
                                 ->hasForeignKey('AuthorId', Author::class)
                                 ->hasOne('Author', Author::class);

                        $builder->entity(Author::class)->hasOne('Picture', Picture::class);
                     }
                  }

Many to Many

Many to many relationships require a collection navigation property on both sides. They will be discovered by convention like other types of relationships. 


                  <?php

                  namespace Application\Models;

                  use Artister\Data\Entity\IEntity;
                  use Artister\system\Collections\IList;

                  class Student implements IEntity
                  {
                     private int $Id;
                     private string $Name;
                     private string $Age;

                     // navigation property
                     private IList $courses;

                     public function __get(string $name){return $this->$name;}

                     public function __set(string $name, $value){$this->$name = $value;}
                  }
               

                  <?php

                  namespace Application\Models;

                  use Artister\Data\Entity\IEntity;
                  use Artister\system\Collections\IList;

                  class Course implements IEntity
                  {
                     private int $Id;
                     private string $Name;

                     // navigation property
                     private IList $Students;

                     public function __get(string $name){return $this->$name;}

                     public function __set(string $name, $value){$this->$name = $value;}
                  }
               

                  <?php

                  namespace Application\Models;

                  use Artister\Data\Entity\IEntity;
                  use Artister\system\Collections\IList;

                  class CourseStudent implements IEntity
                  {
                     private int $StudentId;
                     private int $CourseId;

                     // navigation property
                     private Student $Student;
                     private Course $Course;

                     public function __get(string $name){return $this->$name;}

                     public function __set(string $name, $value){$this->$name = $value;}
                  }

Model configuration: 


                  <?php
                  
                  namespace Application\Models;

                  use Artister\Data\Entity\EntityContext;
                  use Artister\Data\Entity\EntityModelBuilder;
                  use Artister\Data\Entity\EntitySet;

                  class EntityManager extends EntityContext
                  {
                     private EntitySet $Students;
                     private EntitySet $Courses;

                     public function __construct()
                     {
                        $this->Students = $this->Set(Student::class);
                        $this->Courses = $this->Set(Course::class);
                     }

                     public function onModelCreate(EntityModelBuilder $builder)
                     {
                        $builder->entity(StudentCourse::class)
                                 ->hasForeignKey('StudentId', Student::class)
                                 ->hasForeignKey('CourseId', Course::class)
                                 ->hasOne('Student', Student::class)
                                 ->hasOne('Course', Course::class);

                        $builder->entity(Student::class)->hasMany('Categories', PostCategory::class);
                        $builder->entity(Course::class)->hasMany('Posts', PostCategory::class);
                     }
                  }
