Model

Entity ORM model is a conceptual model of an application's domain, made up of entity classes that includes data and a context object reponsible for querying and saving data.

Adding Entity Class

Entity class can be created manually inside "Models" folder or by using DevNet SDK command line: 


devnet add entity --name Student
               

                  <?php

                  namespace Application\Models;

                  use DevNet\Entity\IEntity;

                  class Student implements IEntity
                  {
                     private int $Id;
                     private string $Name;
                     private int $Age;

                     public function __get(string $name)
                     {
                        return $this->$name;
                     }

                     public function __set(string $name, $value)
                     {
                           $this->$name = $value;
                     }
                  }

Note: DevNet SDK will create an empty entity class with one property Id, and you have to define the other properties manualy.

Inisialiation of netity set

adding entity reposotory into the context. 


                  <?php

                  namespace Application\Models;

                  use DevNet\Entity\EntityContext;
                  use DevNet\Entity\EntitySet;
                  use DevNet\Entity\EntityOptions;
                  use DevNet\Entity\EntityModelBuilder;

                  class EntityManager extends EntityContext
                  {
                     private EntitySet $Authors;

                     public function __construct(EntityOptions $options)
                     {
                        parent::__construct($options);
                        $this->Authors = $this->set(Author::class);
                     }

                     public function __get(string $name) : void
                     {
                        return $this->$name;
                     }
                  }

Conventional Configuration

Entity ORM uses a set of conventions to build a model based on the shape of your entity classes, each entity type will be mapped automatically to a database table with the same name as the class name, and each column in the database as the same name of class properties, and by convention when a property named "Id" it will be configured as a primary key of an entity.

API Configuration

You can specify additional configuration to supplement and/or override what was discovered by convention, without modifying your entity classes, by overriding the onModelCreate method in your derived Entitycontext class and use the ModelBuilder API to configure your model. 


                  <?php

                  namespace Application\Models;

                  use DevNet\Entity\EntityContext;
                  use DevNet\Entity\EntityOptions;
                  use DevNet\Entity\EntitySet;
                  use DevNet\Entity\EntityModelBuilder;

                  class EntityManager extends EntityContext
                  {
                     private EntitySet $Authors;

                     public function __construct(EntityOptions $options)
                     {
                        parent::__construct($options);
                        $this->Authors = $this->set(Author::class);
                     }

                     public function __get(string $name) : void
                     {
                        return $this->$name;
                     }

                     public function onModelCreate(EntityModelBuilder $builder)
                     {
                        $builder->entity(Author::class)
                                 ->toTable("posts")
                                 ->hasKey("AuthorId")
                                 ->property("AuthorId")->hasColumn(Author_id);
                     }
                  }

Note: API configuration has the highest precedence and will override conventions and data annotations.
If some table attributes not configured for the given entity, they will be configured by convention.
