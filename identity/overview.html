Overview

Identity Manager is an API that supports user interface login functionality, it manages users, passwords, profile data, roles, claims, tokens, email confirmation, and more.
Users can create an account with the login information stored in Identity, typically is configured using database to store user names, passwords, and profile data. Alternatively.

Registering Identity

The registration of Identity as a service to be used in all the application through dependency injection, is by calling the extension method addIdentity of the IServiceCollection instance, in configure method of Startup class.
Identity service requires other services to work and they are:

  • Authentication
  • Authorization
  • EntityContext


                  <?php
                  
                  namespace Application;
                  
                  use Artister\System\Dependency\IServiceCollection;
                  use Artister\System\Dispatcher\IApplicationBuilder;
                  use Artister\Web\Extensions\DependencyExtensions;
                  use Artister\Web\Extensions\HostingExtensions;
                  use Application\Models\EntityManager;

                  class Startup
                  {
                     public function configureServices(IServiceCollection $services)
                     {
                        $services->addAuthentication();
                        $services->addAuthorisation();

                        $services->addEntityContext(function($options){
                           $options->useConnection("mysql://user:password@127.0.0.1:3306/database");
                           $options->useEntityContext(EntityManager::class);
                        });

                        $services->addIdentity();
                     }
                     ...
                  }

Identity is enabled by calling useAuthentication and useAuthorization, which adds authentication and Authorization middlewares to the request pipeline, and UseRouter, UseAuthentication, UseAuthorization, and UseEndpoint must be called in the order as shown in the following code. 


                  class Startup
                  {
                     ...
                     public function configure(IApplicationBuilder $app)
                     {
                        $app->useRouter();

                        // Authentication middleware must be declared befor Authentication middleware
                        $app->useAuthentication();

                        // Authorisation middleware must be declared befor Endpoint middleware
                        $app->useAuthorization();
                        
                        $app->useEndpoint(function($routes) {
                              $routes->mapRoute("default", "{controller=Home}/{action=Index}/{id?}");
                        });
                     }
                  }
